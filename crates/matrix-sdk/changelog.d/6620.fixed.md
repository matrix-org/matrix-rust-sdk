Accept shared-history room key bundles ([MSC4268](https://github.com/matrix-org/matrix-spec-proposals/pull/4268)) for joins the client did not itself perform. Previously the invite acceptance details were only recorded inside the room-join methods, so if the server joined the user to an invited room (for example by auto-accepting a knock, or when the invite was accepted on another of the user's devices), the inviter's key bundle was silently refused and pre-join history remained undecryptable.
